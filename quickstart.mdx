Get started with Poketto Stack, a production-ready frontend boilerplate built with shadcn/ui, React, TypeScript, and Vite.

<Steps>
  <Step title="Clone the repository">
    Clone the Poketto Stack repository to your local machine:

    ```bash
    git clone https://github.com/TinsFox/poketto-stack.git
    cd poketto-stack
    ```
  </Step>

  <Step title="Install dependencies">
    Poketto Stack uses pnpm as its package manager. Install all dependencies:

    ```bash
    pnpm install
    ```

    <Note>
      Make sure you have pnpm installed. If not, install it with `npm install -g pnpm` or see the [Installation](/installation) guide for more details.
    </Note>
  </Step>

  <Step title="Start the development server">
    Launch the admin application in development mode:

    ```bash
    pnpm admin:dev
    ```

    This runs the Vite development server with React Router. Open [http://localhost:5173](http://localhost:5173) in your browser to see your application.

    <CodeGroup>
    ```bash Admin App
    pnpm admin:dev
    ```

    ```bash All Apps (Monorepo)
    pnpm dev
    ```

    ```bash Tauri Desktop App
    pnpm tauri:dev
    ```
    </CodeGroup>
  </Step>

  <Step title="Explore with Storybook">
    Poketto Stack includes Storybook for developing UI components in isolation:

    ```bash
    cd apps/admin
    pnpm storybook
    ```

    Storybook will open at [http://localhost:6006](http://localhost:6006), where you can browse and interact with all UI components.
  </Step>
</Steps>

## Project Structure

Poketto Stack is organized as a monorepo with the following structure:

```
poketto-stack/
├── apps/
│   ├── admin/          # Main admin application
│   ├── api/            # API service
│   ├── docs/           # Documentation site
│   ├── tauri/          # Tauri desktop app
│   └── website/        # Marketing website
├── packages/
│   ├── ui/             # shadcn/ui components
│   ├── pro-table/      # Advanced table components
│   ├── tiptap/         # Rich text editor
│   └── cli/            # CLI tools
└── package.json        # Root workspace configuration
```

## Create Your First Component

Let's create a simple custom component using the shadcn/ui Button:

<Steps>
  <Step title="Create the component file">
    Create a new file in `apps/admin/app/components/welcome-button.tsx`:

    ```tsx apps/admin/app/components/welcome-button.tsx
    import { Button } from "@poketto/ui/button";

    export function WelcomeButton() {
      return (
        <Button onClick={() => alert("Welcome to Poketto Stack!")}>
          Click me
        </Button>
      );
    }
    ```
  </Step>

  <Step title="Use the component">
    Import and use your component in any route, for example in `apps/admin/app/routes/(main)/index.tsx`:

    ```tsx
    import { WelcomeButton } from "~/components/welcome-button";

    export default function Home() {
      return (
        <div className="flex items-center justify-center min-h-screen">
          <WelcomeButton />
        </div>
      );
    }
    ```
  </Step>

  <Step title="See it in action">
    Your development server should automatically reload. Visit your app to see the new button in action!
  </Step>
</Steps>

## Available Scripts

Poketto Stack provides several npm scripts for common tasks:

| Script | Description |
|--------|-------------|
| `pnpm dev` | Run all apps in development mode |
| `pnpm admin:dev` | Run admin app only |
| `pnpm admin:build` | Build admin app for production |
| `pnpm storybook` | Launch Storybook component explorer |
| `pnpm test` | Run tests in watch mode |
| `pnpm test:coverage` | Run tests with coverage report |
| `pnpm lint` | Lint code with Biome |
| `pnpm format` | Format code with Biome |
| `pnpm check` | Run Biome checks and auto-fix issues |

## Working with UI Components

Poketto Stack includes a comprehensive set of shadcn/ui components in the `@poketto/ui` package:

```tsx Example: Using multiple components
import { Button } from "@poketto/ui/button";
import { Card, CardContent, CardHeader, CardTitle } from "@poketto/ui/card";
import { Input } from "@poketto/ui/input";
import { Label } from "@poketto/ui/label";

export function LoginForm() {
  return (
    <Card className="w-full max-w-sm">
      <CardHeader>
        <CardTitle>Login</CardTitle>
      </CardHeader>
      <CardContent className="space-y-4">
        <div className="space-y-2">
          <Label htmlFor="email">Email</Label>
          <Input id="email" type="email" placeholder="Enter your email" />
        </div>
        <Button className="w-full">Sign In</Button>
      </CardContent>
    </Card>
  );
}
```

<Note>
  All UI components are pre-configured with Tailwind CSS variants and support dark mode out of the box.
</Note>
