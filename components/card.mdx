## Introduction

The Card component provides a structured way to display content in a contained, elevated box. It's perfect for product listings, user profiles, content previews, dashboards, and any UI pattern that benefits from visual grouping.

Cards are composed of multiple sub-components that work together to create a consistent layout structure.

## Installation

```tsx
import {
  Card,
  CardHeader,
  CardTitle,
  CardDescription,
  CardContent,
  CardFooter,
} from "@poketto/ui";
```

## Usage

A basic card with all sections:

```tsx
<Card>
  <CardHeader>
    <CardTitle>Card Title</CardTitle>
    <CardDescription>Card description goes here</CardDescription>
  </CardHeader>
  <CardContent>
    <p>Main content of the card.</p>
  </CardContent>
  <CardFooter>
    <Button>Action</Button>
  </CardFooter>
</Card>
```

## Examples

Minimal card with just content:

```tsx
<Card>
  <CardContent className="pt-6">
    <p>This is a simple card with only content.</p>
```

## Props

### Card

The root container component.

<ParamField path="className" type="string">
  Additional CSS classes. Default styles include rounded corners, border, background, and shadow.
</ParamField>

<ParamField path="children" type="ReactNode">
  Card sub-components or any valid React content.
</ParamField>

Inherits all standard HTML div attributes.

### CardHeader

Container for the card's header section.

<ParamField path="className" type="string">
  Additional CSS classes. Default includes flex column layout with spacing and padding.
</ParamField>

<ParamField path="children" type="ReactNode">
  Typically contains CardTitle and CardDescription.
</ParamField>

### CardTitle

Displays the main heading of the card.

<ParamField path="className" type="string">
  Additional CSS classes. Default includes semibold font, tight leading, and tracking.
</ParamField>

<ParamField path="children" type="ReactNode">
  The title content.
</ParamField>

### CardDescription

Displays supporting text below the title.

<ParamField path="className" type="string">
  Additional CSS classes. Default includes smaller text size and muted foreground color.
</ParamField>

<ParamField path="children" type="ReactNode">
  The description content.
</ParamField>

### CardContent

Main content area of the card.

<ParamField path="className" type="string">
  Additional CSS classes. Default includes padding (with top padding removed if header is present).
</ParamField>

<ParamField path="children" type="ReactNode">
  The main content of the card.
</ParamField>

### CardFooter

Footer section for actions or additional information.

<ParamField path="className" type="string">
  Additional CSS classes. Default includes flex layout, centered items, and padding.
</ParamField>

<ParamField path="children" type="ReactNode">
  Typically contains buttons or action elements.
</ParamField>

## Styling Notes

- All card components are implemented as `div` elements with forwarded refs
- Default card has rounded-xl borders and subtle shadow
- The component uses CSS custom properties for theming via `bg-card` and `text-card-foreground`
- All sub-components accept `className` for easy customization
- Card content and footer have coordinated padding to maintain visual consistency
