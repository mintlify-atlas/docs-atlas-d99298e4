## Introduction

The Separator component provides a simple way to create visual divisions between sections of content. Built on Radix UI's Separator primitive, it supports both horizontal and vertical orientations and includes proper accessibility attributes.

Use separators to improve visual hierarchy and organize content within layouts, menus, lists, and forms.

## Installation

```tsx
import { Separator } from "@poketto/ui";
```

## Usage

By default, the Separator renders a horizontal line:

```tsx
<div>
  <div className="space-y-1">
    <h4 className="text-sm font-medium">Poketto Stack</h4>
    <p className="text-sm text-muted-foreground">Modern full-stack framework</p>
  </div>
  <Separator className="my-4" />
  <div className="space-y-1">
    <h4 className="text-sm font-medium">Features</h4>
    <p className="text-sm text-muted-foreground">Built-in authentication and more</p>
  </div>
</div>
```

## Examples

### Horizontal Separator

Default horizontal divider:

```tsx
<div className="space-y-4">
  <div>
    <h3>Section One</h3>
    <p>Content for the first section.</p>
  </div>
  <Separator />
  <div>
    <h3>Section Two</h3>
    <p>Content for the second section.</p>
  </div>
</div>
```

### Vertical Separator

Use `orientation="vertical"` for vertical dividers:

```tsx
<div className="flex h-5 items-center space-x-4">
  <div>Home</div>
  <Separator orientation="vertical" />
  <div>About</div>
  <Separator orientation="vertical" />
  <div>Contact</div>
</div>
```

### In Navigation

Separate navigation items:

```tsx
<nav className="flex items-center space-x-4">
  <a href="/dashboard">Dashboard</a>
  <Separator orientation="vertical" className="h-4" />
  <a href="/projects">Projects</a>
  <Separator orientation="vertical" className="h-4" />
  <a href="/settings">Settings</a>
</nav>
```

### In Forms

Divide form sections:

```tsx
<form className="space-y-6">
  <div className="space-y-4">
    <h3>Personal Information</h3>
    <Input placeholder="Name" />
    <Input placeholder="Email" />
  </div>
  
  <Separator />
  
  <div className="space-y-4">
    <h3>Account Settings</h3>
    <Input placeholder="Username" />
    <Input type="password" placeholder="Password" />
  </div>
</form>
```

### In Sidebars

Separate menu groups:

```tsx
<aside className="w-64 p-4 space-y-4">
  <div>
    <h4 className="mb-2 font-semibold">Main</h4>
    <ul className="space-y-1">
      <li>Dashboard</li>
      <li>Projects</li>
    </ul>
  </div>
  
  <Separator />
  
  <div>
    <h4 className="mb-2 font-semibold">Settings</h4>
    <ul className="space-y-1">
      <li>Profile</li>
      <li>Preferences</li>
    </ul>
  </div>
</aside>
```

### Custom Styling

Customize appearance with className:

```tsx
<div>
  <p>Content above</p>
  <Separator className="my-8 bg-gradient-to-r from-transparent via-primary to-transparent" />
  <p>Content below</p>
</div>
```

## Props

<ParamField path="orientation" type="'horizontal' | 'vertical'" default="horizontal">
  The orientation of the separator. Horizontal creates a horizontal line (1px height, full width), while vertical creates a vertical line (full height, 1px width).
</ParamField>

<ParamField path="decorative" type="boolean" default={true}>
  When `true`, the separator is purely decorative and hidden from screen readers. When `false`, it's announced to screen readers as a separator.
</ParamField>

<ParamField path="className" type="string">
  Additional CSS classes to customize the separator appearance. The component automatically applies appropriate dimensions based on orientation.
</ParamField>

Inherits all props from Radix UI Separator primitive.

## Styling

The Separator uses the following default styles:

- **Background**: Uses `bg-border` theme color
- **Horizontal**: `h-[1px] w-full`
- **Vertical**: `h-full w-[1px]`
- **Shrink**: Set to `shrink-0` to prevent flex shrinking

You can override any of these with the `className` prop.

## Accessibility

- Built on Radix UI Separator following WAI-ARIA separator pattern
- The `decorative` prop defaults to `true`, making the separator invisible to screen readers
- Set `decorative={false}` when the separator provides meaningful structure
- Proper ARIA roles and attributes are automatically applied
- No interactive behavior or keyboard focus required
