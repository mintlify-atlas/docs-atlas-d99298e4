The Chart component provides a set of composable charting primitives built on top of Recharts, with automatic theme support and responsive containers.

## Installation

```bash
npm install @poketto/ui recharts
```

## Import

```tsx
import {
  ChartContainer,
  ChartTooltip,
  ChartTooltipContent,
  ChartLegend,
  ChartLegendContent,
} from "@poketto/ui";
import { Bar, BarChart, Line, LineChart, XAxis, YAxis } from "recharts";
```

## Basic Usage

```tsx
const chartData = [
  { month: "Jan", revenue: 4000 },
  { month: "Feb", revenue: 3000 },
  { month: "Mar", revenue: 5000 },
  { month: "Apr", revenue: 4500 },
  { month: "May", revenue: 6000 },
];

const chartConfig = {
  revenue: {
    label: "Revenue",
    color: "hsl(var(--chart-1))",
  },
};

<ChartContainer config={chartConfig}>
  <BarChart data={chartData}>
    <Bar dataKey="revenue" fill="var(--color-revenue)" />
  </BarChart>
</ChartContainer>
```

## Chart Configuration

The `ChartConfig` object defines the visual and semantic properties of your chart data:

```tsx
const chartConfig = {
  revenue: {
    label: "Revenue",
    color: "hsl(220, 70%, 50%)",
  },
  expenses: {
    label: "Expenses",
    theme: {
      light: "hsl(0, 70%, 50%)",
      dark: "hsl(0, 70%, 60%)",
    },
  },
  profit: {
    label: "Profit",
    color: "hsl(142, 70%, 50%)",
    icon: TrendingUp,
  },
};
```

## Examples

### Line Chart

```tsx
import { LineChart, Line, XAxis, YAxis, CartesianGrid } from "recharts";

const data = [
  { name: "Jan", users: 400 },
  { name: "Feb", users: 300 },
  { name: "Mar", users: 600 },
  { name: "Apr", users: 800 },
  { name: "May", users: 500 },
];

const config = {
  users: {
    label: "Users",
    color: "hsl(var(--chart-1))",
  },
};

<ChartContainer config={config}>
  <LineChart data={data}>
    <CartesianGrid strokeDasharray="3 3" />
    <XAxis dataKey="name" />
    <YAxis />
    <ChartTooltip content={<ChartTooltipContent />} />
    <Line 
      type="monotone" 
      dataKey="users" 
      stroke="var(--color-users)" 
      strokeWidth={2}
    />
  </LineChart>
</ChartContainer>
```

```tsx
const data = [
  { month: "Jan", revenue: 4000, expenses: 2400 },
  { month: "Feb", revenue: 3000, expenses: 1398 },
  { month: "Mar", revenue: 5000, expenses: 3800 },
  { month: "Apr", revenue: 4500, expenses: 3908 },
];

const config = {
  revenue: {
    label: "Revenue",
    color: "hsl(var(--chart-1))",
  },
  expenses: {
    label: "Expenses",
    color: "hsl(var(--chart-2))",
  },
};

<ChartContainer config={config}>
  <BarChart data={data}>

  </BarChart>
```


## Component API

### ChartContainer

The root container that provides context and responsive sizing.

**Props:**

- `config`: `ChartConfig` - Configuration object defining chart colors and labels
- `id`: `string` (optional) - Custom ID for the chart
- `className`: `string` (optional) - Additional CSS classes
- `children`: Recharts components

### ChartConfig

Configuration type for defining chart data series:

```tsx
type ChartConfig = {
  [key: string]: {
    label?: React.ReactNode;
    icon?: React.ComponentType;
  } & (
    | { color?: string; theme?: never }
    | { color?: never; theme: { light: string; dark: string } }
  );
};
```

### ChartTooltip

Recharts Tooltip component (re-exported).

**Props:** See [Recharts Tooltip documentation](https://recharts.org/en-US/api/Tooltip)

### ChartTooltipContent

Custom tooltip content with automatic theming.

**Props:**

- `hideLabel`: `boolean` - Hide the tooltip label
- `hideIndicator`: `boolean` - Hide the color indicator
- `indicator`: `"line"` | `"dot"` | `"dashed"` - Indicator style (default: `"dot"`)
- `nameKey`: `string` - Key for the data name
- `labelKey`: `string` - Key for the label
- `labelFormatter`: `(value, payload) => React.ReactNode` - Custom label formatter
- `formatter`: Custom value formatter

### ChartLegend

Recharts Legend component (re-exported).

**Props:** See [Recharts Legend documentation](https://recharts.org/en-US/api/Legend)

### ChartLegendContent

Custom legend content with automatic theming.

**Props:**

- `hideIcon`: `boolean` - Hide the legend icons
- `nameKey`: `string` - Key for the data name
- `payload`: Legend payload from Recharts
- `verticalAlign`: `"top"` | `"bottom"` - Vertical alignment

### useChart

Hook to access chart configuration context.

```tsx
const { config } = useChart();
```

## Advanced Usage

### Custom Tooltip

```tsx
<ChartContainer config={chartConfig}>
  <LineChart data={data}>
    <ChartTooltip
      content={
        <ChartTooltipContent
          indicator="line"
          labelFormatter={(value) => `Date: ${value}`}
        />
      }
    />
    <Line dataKey="value" stroke="var(--color-value)" />
  </LineChart>
</ChartContainer>
```

### Theme-Specific Colors

```tsx
const config = {
  revenue: {
    label: "Revenue",
    theme: {
      light: "hsl(220, 70%, 50%)",
      dark: "hsl(220, 70%, 70%)",
    },
  },
};
```

### Custom Aspect Ratio

```tsx
<ChartContainer config={config} className="aspect-square">
  <BarChart data={data}>
    <Bar dataKey="value" />
  </BarChart>
</ChartContainer>
```

### With Icons in Legend

```tsx
import { TrendingUp, TrendingDown } from "lucide-react";

const config = {
  profit: {
    label: "Profit",
    color: "hsl(142, 70%, 50%)",
    icon: TrendingUp,
  },
  loss: {
    label: "Loss",
    color: "hsl(0, 70%, 50%)",
    icon: TrendingDown,
  },
};
```

## Recharts Integration

This component uses Recharts under the hood. All Recharts components are compatible:

- `BarChart`, `LineChart`, `AreaChart`, `PieChart`, `RadarChart`, etc.
- `XAxis`, `YAxis`, `CartesianGrid`, `Legend`, `Tooltip`
- `Bar`, `Line`, `Area`, `Pie`, `Radar`, etc.

See the [Recharts documentation](https://recharts.org/) for complete API reference.

## Theming

The Chart component automatically adapts to your theme:

- Uses CSS variables for colors (`--chart-1`, `--chart-2`, etc.)
- Supports light/dark mode through theme configuration
- Automatic color scheme for grid lines, axes, and text
- Customizable via Tailwind CSS classes

## Accessibility

- Responsive container automatically sizes charts
- Semantic SVG output from Recharts
- Tooltip provides context for data points
- Consider adding ARIA labels for screen readers
