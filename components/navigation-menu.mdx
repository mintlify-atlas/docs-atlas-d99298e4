Navigation Menu provides a structured way to organize primary navigation with support for dropdown content panels.

## Installation

```tsx
import {
  NavigationMenu,
  NavigationMenuList,
  NavigationMenuItem,
  NavigationMenuTrigger,
  NavigationMenuContent,
  NavigationMenuLink,
  NavigationMenuIndicator,
  NavigationMenuViewport,
  navigationMenuTriggerStyle,
} from "@poketto/ui";
```

## Basic Usage

```tsx
<NavigationMenu>
  <NavigationMenuList>
    <NavigationMenuItem>
      <NavigationMenuTrigger>Getting Started</NavigationMenuTrigger>
      <NavigationMenuContent>
        <ul className="grid gap-3 p-6 md:w-[400px] lg:w-[500px]">
          <li>
            <NavigationMenuLink asChild>
              <a href="/docs/installation">Installation</a>
            </NavigationMenuLink>
          </li>
          <li>
            <NavigationMenuLink asChild>
              <a href="/docs/introduction">Introduction</a>
            </NavigationMenuLink>
          </li>
        </ul>
      </NavigationMenuContent>
    </NavigationMenuItem>
    <NavigationMenuItem>
      <NavigationMenuLink href="/about" className={navigationMenuTriggerStyle()}>
        About
      </NavigationMenuLink>
    </NavigationMenuItem>
  </NavigationMenuList>
</NavigationMenu>
```

## Component API

### NavigationMenu

Root component that provides context for all menu items.

**Props:**
- Extends `@radix-ui/react-navigation-menu` Root props
- `value?: string` - Controlled active item value
- `onValueChange?: (value: string) => void` - Value change handler
- `defaultValue?: string` - Default active value
- `delayDuration?: number` - Hover delay in ms (default: 200)
- `skipDelayDuration?: number` - Skip delay after interaction (default: 300)

**Note:** The NavigationMenu component automatically includes a NavigationMenuViewport at the end.

### NavigationMenuList

Container for menu items.

**Props:**
- Extends `@radix-ui/react-navigation-menu` List props

### NavigationMenuItem

Individual menu item container.

**Props:**
- Extends `@radix-ui/react-navigation-menu` Item props
- `value?: string` - Unique identifier for the item

### NavigationMenuTrigger

Button that toggles the content panel.

**Props:**
- Extends `@radix-ui/react-navigation-menu` Trigger props
- Automatically includes a ChevronDown icon that rotates on open

### NavigationMenuContent

Container for dropdown content.

**Props:**
- Extends `@radix-ui/react-navigation-menu` Content props
- Includes smooth animations for enter/exit

### NavigationMenuLink

Navigational link element.

**Props:**
- Extends `@radix-ui/react-navigation-menu` Link props
- `asChild?: boolean` - Compose with child element
- `active?: boolean` - Active state styling

### NavigationMenuViewport

Portal viewport for content panels.

**Props:**
- Extends `@radix-ui/react-navigation-menu` Viewport props
- Automatically rendered by NavigationMenu root component

### NavigationMenuIndicator

Visual indicator showing active item.

**Props:**
- Extends `@radix-ui/react-navigation-menu` Indicator props
- Renders a small arrow/pointer below the active trigger

### navigationMenuTriggerStyle

Helper function that returns trigger styling classes.

**Usage:**
```tsx
<NavigationMenuLink className={navigationMenuTriggerStyle()}>
  Link
</NavigationMenuLink>
```

## Composition Pattern

The NavigationMenu component follows this structure:

```
NavigationMenu
  ├─ NavigationMenuList
  │   ├─ NavigationMenuItem
  │   │   ├─ NavigationMenuTrigger
  │   │   └─ NavigationMenuContent
  │   │       └─ NavigationMenuLink (multiple)
  │   ├─ NavigationMenuItem
  │   │   └─ NavigationMenuLink
  │   └─ NavigationMenuIndicator (optional)
  └─ NavigationMenuViewport (auto-included)
```

## Accessibility

- Built on Radix UI primitives with full ARIA support
- Keyboard navigation with arrow keys between triggers
- Tab/Shift+Tab to navigate through content
- Escape to close open content
- Enter/Space to activate triggers
- Automatic focus management when content opens
- `aria-current` attribute on active links
- Proper focus indicators for all interactive elements
- ChevronDown icon has `aria-hidden="true"` to avoid screen reader confusion
- Smooth animations respect `prefers-reduced-motion`
